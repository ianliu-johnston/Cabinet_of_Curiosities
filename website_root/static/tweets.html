Lessons to Avoid Ruin

Dangerous Batch Script

I made this error before I went more in depth with sed. As I was trying to write a batch script to change all of my nav bar titles, I accidentally created a script that changed every single word into "Nothingness." It was a perfect word to replace all of the others with. Fortunately, I kept a current backup (like a good coder) and was able to continue on. I kept the file because I thought it was funny. This is it below:

This seemingly endless stream of "Nothingness" is the result of a poorly constructed batch script.

You can find the page here, it's still in a text document on the server for reference, or humor.

It's pretty funny to view the source files, because it didn't replace the punctuation, just everything else. In case you're interested, the batch script I created was something along the lines of

  • sed -i 's/!&lt;&lt;&#9760;&gt;&gt;!/Mistakes/g' mistakes.html
  • sed -i 's/((\/\/&#8578;\\\\))/Home/g' home.html
  • sed -i 's/!!&#9156;&#8734;&#9156;!!/Results/g' results.html
  • sed -i 's/((\/\/&#8710;\\\\))/Chaos/g' chaos.html
  • sed -i 's/&#9806;||&#9795;\\\\&#9806;/Nothingness/g' nothingness.html

These were my original menu buttons(which was also a mistake):

I think I missed some escape characters or a closing parenthesis in the funky file. I was trying to find a better way to change all of the HTML files at once, than by hand, or copy-pasting the content back into the articles. Also, an easier way to manipulate text files outside of a text-editing program. Then I discovered sed and head. It was pretty easy to isolate the items in the navbar to be changed, because they are a unique string of characters. Eventually the script above actually worked.

Escape Characters and HTML Entities

When I ran my website through the html validator I discovered that the "<>" tags are reserved in HTML documents, because they are used to deliniate between normal text and the end of a tag. They're called HTML Entities." Instead of just typing shift + , on your keyboard, you write &lt; and &gt;
