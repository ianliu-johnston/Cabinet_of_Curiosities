Path and Progressions

Intro:

This website is made for the admission process. I started out the process of making this website with a little bit of pre-knowledge of how to code in HTML, but along the way learned a ton about the process. Coding takes patience, especially through the command line. Things are not quite what you think they will be initially, but the more you poke around, the more you can adapt to the mindset it takes to figure things out.

Building From The Command Line:

Basics:

I saw this website as a challenge to get to know the Linux environment better, so I built this website entirely from within the Command Line Interface on my remote server. I did not use any text editors on my local machine. This means that I had to figure out how to do all of it without a Graphical User Interface. I used commands like wget http://example.com, sed (see below), emacs file.html, head -n 25 file.html > header.txt, echo some text >> file.html, touch emptyFile.html and lots of ssh, cd, ls, cp, mkdir, rm -r directory and rm. Some of the images, I had to edit on my local machine and upload to the remote server with scp picture.png root@thewebsite/filepath.

Backups:

This was a crucial element of the process. I ended up making a series of archives that backed up my website

These are the steps that I took to create them:

  1. Make a folder called bkups in /var/www/ with this command: mkdir bkups.
  2. Create a batch script to copy all Javascript, CSS, and HTML files into it:

    echo cp *.html *.css *.js >> backItUp.sh

    echo tar -zcvf bkupArchive.tar.gz bkups >> backItUp.sh

  3. Make the batch script executable: chmod +x backItUp.sh
  4. Run it when I need to backup: ./backItUp.sh

The sed Command:

Pretty far along in the process, I discovered an easier way to modify all of the documents on my HTML site. This marvelous utility simply called 'sed'. (The link in the previous headline is to a tutorial about sed.) I kept wanting to change the navigation bar to include the new pages, or delete the random ones, and I didn't want to have to go back and hand type it every time. Even with the template file I generated, I still had to copy paste the content back in. Tedium. After a little bit of googling, I found this command:

sed -i 's/patternToReplace/ReplacingPattern/g' file.html

I tried it out on a copied template.html file, and viola! It changed just that one line to be correct. I tried listing more files like this:

sed -i 's/pattern/replacement/g' file1.html file2.html file3.html file4.html file5.html

That worked too! Now I had a functional command, but there were pitfalls to this method too. See this page for some mistakes I made.

Here are the different permutations of the sed command I used:

sed -i 's/pattern/replacement/g' file.html file2.html #replaces a pattern with another pattern

sed -i '25i<li>List item to insert at line number 25</li>' file.html #inserts a pattern at a line specified by __yourLineNumber and i__

sed -i '/pattern to delete/d' file.html #deletes a pattern

sed -i '26d' file.html #deletes line number 26

sed -n '18,24 p' file.html #displays lines 18-24 from the file. Does not modify them

Here is a webpage to some helpful commands.

Note that the -i wildcard changes the files permanently, whereas omitting the -i wildcard prints the result to the output.
